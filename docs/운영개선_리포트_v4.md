## 0. 한 줄 결론
v3 트리거를 실전 가동하자 두 가지 중대 버그가 드러남: (1) 매집 스파이크를 "하루 전체 최대"로 잡아 14시간 전 매집이 밤까지 유령 신호로 발송됨 → 최근 4시간 필터로 해결. (2) verifier가 4시간 보유한계를 넘겨서까지 TP를 판정 → 성과가 과대평가됨 → 4h 엄격 적용 + 고가/저가 정확 판정으로 수정. 그리고 원본 알파 IN을 우리 봇이 선점 포착(+13%)한 것이 실시간 입증됨.

## 1. 실시간 검증 — IN 원본 알파 선점 (하이라이트)
원본 채널 "차트에 미친사람들"이 IN(인피닛) 127원 @19:30 신호 발송, 수익률 13~15% 주장.
**우리 봇은 IN을 두 번, 모두 원본보다 먼저 포착:**
- 13:52 진입 114원 → 4h내 +13.2% (원본보다 5h38m 선점, 진입가 13원 낮음)
- 19:20 진입 124원 → +12.9% (원본보다 10분 선점)
1틱 매집 스파이크 기준 IN은 전종목 250개 중 #2위(상위 0.8%)로 동적컷 안에 충분히 포함.
→ 새 트리거(1틱+동적컷)가 원본 알파를 따라가는 게 아니라 **선점**함을 실증.

## 2. 버그 ① — 유령 신호 (매집 스파이크 하루 유지)
**증상:** 한밤중에 "매집 @09:27 ⏰장초반" 알람 도착. 09:27 매집이 14시간 후 발송됨.
**원인:** collector가 하루 전체 CSV에서 최대 스파이크를 찾음 → 오전 큰 매집이 그날 종일 후보 상위에 박혀, 수축 조건만 맞으면 언제든 신호.
**수정:** SPIKE.MAX_AGE_HOURS=4 — 스파이크 시각이 최근 4시간 이내인 매집만 후보.
**검증:** CSV 최신(21:34) 기준 17시 이전 매집 0개로 전부 제외 확인.
**부수효과:** 장초반(07~11시) 라벨이 4h 필터와 맞물려 15시 이후 자연 소멸 (오래된 장초반 매집은 만료).

## 3. 버그 ② — verifier 성과 과대평가 (사용자 지적)
**증상:** signals_log에 IN 13:52가 "MFE 13.16% / tp3"로 기록. 그러나 WEAK TP3=+15%인데 +13%로 미달 → 모순.
**원인 1:** 4시간 경과 후에도 TP 판정 (시간체크가 _splitExit 이후) → 보유한계(17:52) 넘긴 19:30 발사 구간 가격으로 tp3 청산.
**원인 2:** 5분봉 종가 1개만 봄 → 봉내 고가 누락, MFE/TP 부정확.
**수정:**
- 4시간 경과 시 TP 판정 없이 즉시 종료 (보유한계 엄격)
- MFE=고가, MAE=저가 (봉내 최고/최저 포착)
- TP는 고가, 손절은 저가 기준 (같은 봉 동시 터치 시 손절 우선 = 보수적)
- 청산사유에 time_tp1/time_tp2 추가 (시간초과여도 도달 TP 반영)
**재계산:** IN 13:52 → 올바르게 "MFE 13.16% / time_tp2" (tp3 아님). 실현 +7.43%.
**의의:** signals_log가 성과 판단의 기준이므로, 이 버그가 있으면 모든 TP 도달률·백테스트 결론이 흔들림. 측정 신뢰성 자체를 복구.

매도상태 🟠 보통 (매도 우위)        ← 🔥강한마름/🟠보통 (강중 중복 제거)

─────────────
📋 현재가 / 🛑 손절 / 🎯 TP1~3 (국면별)

- 강/중 라벨이 매도상태와 중복 → 제거하고 매도상태에 🔥/🟠 통합
