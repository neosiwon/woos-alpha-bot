# 틱 봇벽 탐지 시스템 (woos_wall_detector)

> 작성: 2026-05-30 | 프로젝트: woos-alpha-bot (단타/알파스캐너)
> 목적: 일봉에 묻히는 세력 매집/분배를 틱(체결단위) 해상도로 포착 → 텔레그램 알람

---

## 0. 배경 — 왜 틱인가

- **단타 종목선정 = 일봉/분봉/60초로는 불가** 확정 (4각도 검증, 변별력 0).
  세력 매집이 집계에 묻혀서 "거래 죽음"으로만 보임.
- 핵심 논리: **세력이 있으면 매집은 반드시 체결로 남는다.** 일봉에 안 보일 뿐.
- → 비집계 데이터인 **틱(체결 한 건 단위)** 이 마지막 미검증 영역.

## 1. 시스템 구성

```
woos-tick-collector.service   업비트 전종목 trade WS → 활성종목 raw 틱 저장
  └ ~/woos_logs/tick_YYYYMMDD.csv.gz (시각·종목·체결가·체결량·체결액·BID/ASK·체결ID)

woos-wall.timer (1시간마다)
  └ woos_wall_detector.py → 봇벽 판정
       ├ ~/woos_logs/pattern_YYYYMMDD.csv (기록)
       ├ 텔레그램 모니터방 발송 (🟢매집 / 🔴분배)
       └ ~/woos_logs/wall_state.json (N차 카운트 + dedup)
```

## 2. 봇벽 판정 기준

한 방향 봇 벽 = 아래 4개 동시 충족:

| 조건 | 임계값 | 의미 |
|---|---|---|
| 최빈 체결액 반복률 | ≥ 15% | 같은 금액이 계속 체결 |
| 한 방향 쏠림 | ≥ 90% | 매수 or 매도 한쪽 |
| 한 가격 집중 | ≥ 70% | 특정 가격에 박힘 |
| 중앙 체결간격 | ≤ 8초 | 봇/알고 (사람 아님) |

- 추가: 최소 틱 50건, 최빈금액 최소 15회, 메이저(BTC/ETH/XRP) 제외
- **매수벽** = 매집 정황 (받치기) / **매도벽** = 분배 정황 (누르기)

## 3. 실증 사례 (2026-05-30 첫 탐지)

| 종목 | 방향 | 반복금액 | 횟수 | 쏠림 | 가격 | 간격 | 가격변화 |
|---|---|---|---|---|---|---|---|
| KAT | 매수벽 | 49,000원 | 39회 | 100% | 10.5원 | 3초 | +0.0% |
| SAHARA | 매도벽 | 203,000원 | 131회 | 99% | 48.0원 | 2초 | -0.4% |

- **KAT**: 10.5원에 49000원씩 2~3초 간격 100% 매수 → 가격 0.0% (받치기/매집 방어)
- **SAHARA**: 48원에 203000원씩 2초 간격 99% 매도 → 분배 (이미 +116% 폭등 후)
- ★ **둘 다 일봉상 가격 거의 무변(0.0%, -0.4%)** = 일봉엔 안 보임. 틱에선 선명.
  → "세력 있으면 매집 체결 남는다"의 첫 실증.

## 4. 텔레그램 알람 형식

```
🟢 매집 정황 감지 — 2차
종목: KAT
매수벽 (받치기): 49,000원 × 39회 (100% 쏠림)
가격: 10.5원 고정 (집중 95%)
간격: 3초 (봇/알고)
오늘 KAT 매수벽 2번째
```

- N차 카운트: 같은 종목+방향 반복 시 증가 (세력 의지 강도). 원본채널 "오늘 N번째" 구조.
- 발송처: TELEGRAM_CHAT_ID_MONITOR (개인 모니터방, 단톡방 X — 노이즈 분리)

## 5. 한계 / 검증 예정

- **한계**: 수집기가 "활성(급증)" 종목만 raw 저장 → 발사 전 조용한 매수벽은 raw 자체가 없어 놓칠 수 있음. (관심종목 상시저장 모드가 다음 후보)
- **검증 예정**: 며칠 누적 후
  - 매수벽 뜬 종목 → 며칠 뒤 오르나? (매수벽 = 발사 선행 신호인가)
  - 매도벽 뜬 종목 → 빠지나? (매도벽 = 분배 신호인가)
  - KAT 첫 테스트케이스 (매수벽 떴으니 추적)

## 6. 운영 메모

- 파일 교체: GCP SSH 업로드는 동명파일을 (1)(2)로 저장하므로 base64 deploy 또는 mv로 교체.
- state 초기화(재테스트): `rm ~/woos_logs/wall_state.json`
- 수동 실행: `python3 ~/woos_wall_detector.py [YYYYMMDD]` (인자 없으면 최신 틱)
- .env 키: TELEGRAM_BOT_TOKEN, TELEGRAM_CHAT_ID_MONITOR
