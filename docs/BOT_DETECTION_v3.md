# 수량고정 봇 탐지 + 매집→표류→신호 메커니즘 (v3)

> 작성: 2026-05-30 | 프로젝트: woos-alpha-bot (단타/알파스캐너)
> 핵심: 세력 봇을 '수량 고정'으로 식별, 매집→표류→신호 메커니즘 규명, 5분 주기 알람

---

## 0. 배경 — 종목선정 돌파 과정

- 단타 종목선정 = 일봉/분봉/60초로 불가 확정 (변별력 0, 4각도 검증).
- 세력 매집이 집계에 묻혀 "거래 죽음"으로만 보임 → 틱(체결 단위)이 마지막 영역.
- 틱 봇벽 탐지 v1: "동일금액 + 한방향 90%" 기준 → KAT·SAHARA 잡음.
- **문제 발견**: ID 신호 종목 분석 중 — 진짜 세력 봇은 양방향이라 v1이 놓침.

## 1. ★ 봇 식별 = 수량 고정 (방향 아님)

세력 봇의 진짜 지표는 **체결 수량이 소수점까지 동일하게 반복**되는 것.

| 항목 | 설명 |
|---|---|
| 핵심 | 최빈 체결수량이 소수점 3자리+ 동일 + 수회 반복 = 봇 (사람 불가) |
| 금액 ≠ 지표 | 금액이 어중간(77,632원)한 건 수량고정이라 가격따라 나온 결과 |
| 방향 다양 | 매수=매집 / 매도=분배 / 양방향=중립. 한 방향으로 거르면 안 됨 |
| 잡음 제거 | 정수·라운드 수량(5,100,135개)은 사람도 우연히 맞춤 → 제외 |

**실증 (2026-05-30 14시 실시간):**
- KAT: 45,871.56개 × 42회, 100% 매수 = 매집 (어제 49,000원→오늘 50만원 지속)
- LAYER: 744.00개 × 29회, 100% 매수 = 매집 (방금 시작)
- MIRA: 657.90개 × 29회, 100% 매도 = 분배
- POKT: 277.174개 × 97회 = 매집 (원본 신호 종목)
- MOVE: 225.664개 × 93회 = 매집

소수점 어중간한 수량이 수십 회 정확히 반복 = 봇 확정. 사람은 불가능.

## 2. ★★ 매집 → 표류 → 신호 메커니즘 (ID 실증)

원본 신호 종목 ID(5/30 10:44 신호, +35%)의 신호 전 틱 분석으로 규명.

```
05~09:33  세력 봇 매집 (매도 섞어 누르며 흡수, 가격 54→52)
09:33     매집 완료 → 봇 멈춤
          ★ 거래 17,614건 → 119건 (148배 급감, 99.3%↓)
          ★ 가격 52.2원 박제 (표류 = 방치)
10:44     원본 신호 (조용한 표류 구간 = "거래 죽음")
11시~     세력 매수 전환 → 가격 올림 (52→74, +35%)
```

**핵심 통찰:** "거래 죽음"은 자연스러운 조용함이 아니라,
**세력이 매집을 끝내고 만든 인위적 정적**이다.
- 우리가 검증한 원본 신호 20건 중 19건이 "거래 죽음+횡보"였던 이유 = 이것.
- 매집 끝 → 거래 뚝 → 표류 → 원본이 이 정적을 신호로 잡음 → 발사.
- 즉 거래죽음 = 매집 완료의 신호 (결과지 원인 아님).

## 3. 검증 가설 (데이터 누적 중)

**"매집 종료 → 약 1시간 뒤 상승/신호"**
- ID: 매집끝 09:33 → 신호 10:44 (약 1시간 11분)
- 1건이라 패턴 확정은 미정. 전종목 상시저장으로 추가 검증 예정.
- 1시간 시차면 분단위 정밀도 불필요 → 5분 주기 탐지로 충분.

## 4. 봇벽 탐지기 v3 — 매집 포착 / 종료 2단 알람

```
탐지: 5분마다, 최근 15분 윈도우
판정: 수량 최빈값이 소수점 3자리+ & 4회+ & 간격 30초 이내 & 라운드 아님
방향: 매수≥70%=매집 🟢 / 매도≥70%=분배 🔴 / 섞임=중립 ⚪

[알람1 — 매집 포착]
  수량고정 봇 새로 감지 (직전 회차에 없던 종목)
  하루 1번 dedup (5분마다 중복 방지)

[알람2 — 매집 종료] ★신호 선행 후보
  직전 매집봇이 사라짐 + 거래 70%+ 급감 (둘 다)
  → "표류 시작 = 신호 선행 자리"
  → 이후 약 1시간 관찰 (가설 검증)
```

발송처: TELEGRAM_CHAT_ID_MONITOR (개인 모니터방).

## 6. 알파봇 알림 차단 (2026-05-30)

- 거래죽음/체결강도 신호 봇 = "거래급증 때 진입 = 원본과 정반대 = 손실" 검증됨.
- → 무의미 + 모니터방 노이즈 → notify.js `_send`에 return 삽입으로 발송 차단.
