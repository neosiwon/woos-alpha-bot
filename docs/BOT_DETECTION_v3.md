# 수량고정 봇 탐지 + 매집→표류→신호 메커니즘 (v3)

> 2026-05-30 | woos-alpha-bot (단타/알파스캐너)
> 세력 봇을 '수량 고정'으로 식별, 매집→표류→신호 메커니즘 규명, 5분 탐지 + 4h/하루 요약

## 1. 봇 식별 = 수량 고정 (방향 아님)
세력 봇의 진짜 지표 = 체결 수량이 소수점까지 동일하게 반복.
- 핵심: 최빈 체결수량 소수점 3자리+ 동일 & 수회 반복 = 봇 (사람 불가)
- 금액(77,632원 등 어중간)은 수량고정이라 가격따라 나온 결과 — 금액은 지표 아님
- 방향 다양: 매수=매집 / 매도=분배 / 양방향=중립. 한 방향으로 거르면 ID 같은 봇 놓침
- 잡음 제거: 정수·라운드 수량(5,100,135개)은 제외 (사람도 우연히 맞춤)

실증(5/30): KAT 45,871.56×42 매수=매집 / MIRA 657.90×29 매도=분배 /
LAYER 744.00×29 매수 / POKT 277.174×97 매집(원본신호종목) / MOVE 225.664×93

## 2. 매집→표류→신호 메커니즘 (ID 실증)
ID 5/30 10:44 신호(+35%) 신호前 틱 분석:
```
05~09:33  세력봇 매집 (매도 섞어 누르며 흡수, 54→52)
09:33     매집완료 봇멈춤 → 거래 17,614→119건 (148배 급감) + 가격 52.2 박제(표류)
10:44     원본 신호 (조용한 표류 = "거래죽음")
11시~     매수 전환 → +35%
```
★"거래죽음" = 자연스런 조용함 아니라 **세력이 매집 끝내고 만든 인위적 정적**.
원본신호 20건중 19건 거래죽음+횡보였던 이유 = 매집완료의 결과(원인 아님).
★매집은 매도우위로 나타남 (누르며 흡수). 순매도라고 매집 아닌 게 아님.

## 3. 가설 (검증중)
매집종료 → 약 1시간 뒤 상승 (ID 09:33→10:44=1h11m). 1건이라 추가검증.
1시간 시차면 5분 탐지로 충분 (분단위 정밀 불필요).

## 4. 봇벽 탐지기 v3 (woos_wall_detector.py, 5분 주기)
판정: 수량 소수점3자리+동일 & 4회+ & 비율10%+ & 평균체결액 4만원+(KAT 49000 통과) &
간격30초 & 라운드수량 제외 & 메이저 제외. 윈도우 15분.
방향: 매수70%+ 매집🟢 / 매도70%+ 분배🔴 / 섞임 중립⚪ (다 알람, 전환시 재알림)

알람 2단: [매집포착] 새 봇 / [매집종료] 봇 사라짐+거래70%급감=표류시작=신호선행
